# Accessible School Navigator

휠체어 사용자가 학교 본관 안에서 계단과 접근하기 어려운 통로를 피해 이동 경로를 찾을 수 있는 프론트엔드 내비게이션입니다. 프로젝트의 1층·2층·3층 학교 안내 이미지에서 **본관으로 표시된 구역만** 추려 그래프로 모델링하고, 지하철 노선도처럼 단순화한 SVG 지도로 보여 줍니다. `1-1`부터 `3-5`까지 각 학급을 독립된 목적지로 제공하며, 안내도의 좌우 순서와 복도 위·아래 위치를 반영합니다.

## 주요 기능

- 층별로 그룹화된 출발지·목적지 선택
- 일반 이동 / 휠체어 이동 모드
- 직접 구현한 Dijkstra 최단 경로 탐색
- 계단, 장애물, 좁은 통로, 급경사 접근성 필터
- 1–3층 SVG 위상 지도와 추천 경로 강조
- 경로가 있는 층 표시 및 층별 전환
- 총비용, 이동 단계, 수직 이동 수단, 접근성 점수 요약
- 현재 단계 강조와 이전·다음 단계 시뮬레이션
- Web Speech API 기반 한국어 음성 안내
- 키보드 조작, 포커스 표시, ARIA 상태 안내 및 반응형 UI

## 사용 기술

React, TypeScript, Vite, SVG, CSS, Web Speech API, Vitest를 사용합니다. 외부 백엔드나 지도 API는 사용하지 않습니다.

## 설치 및 실행

Node.js 20 이상을 권장합니다.

```bash
npm install
npm run dev
```

터미널에 표시되는 로컬 주소를 브라우저에서 엽니다. 프로덕션 빌드와 테스트는 다음과 같습니다.

```bash
npm run build
npm test
```

## 데이터 수정 방법

- 장소(노드): `src/data/nodes.ts`
- 통로(간선): `src/data/edges.ts`
- 타입: `src/types/navigation.ts`
- 접근성 기준과 비용 상수: `src/algorithms/dijkstra.ts`

### 노드 추가

`navigationNodes` 배열에 고유한 `id`, 표시 이름, 층, SVG 좌표, 유형을 가진 객체를 추가합니다. `x`, `y`는 실제 좌표가 아니라 `0 0 1000 700` SVG 안에서의 시각적 위치입니다.

```ts
{ id: 'music-2', name: '음악실', floor: 2, x: 180, y: 400, type: 'room' }
```

### 간선 추가

`navigationEdges` 배열에 고유한 `id`와 존재하는 두 노드 ID를 연결하는 객체를 추가합니다. `bidirectional`을 생략하면 역방향 이동도 허용됩니다. 일방통행은 `false`로 설정합니다.

```ts
{
  id: 'e26', from: 'hall-2', to: 'music-2', distance: 12,
  accessible: true, type: 'corridor', width: 1.3,
  instruction: '중앙 복도를 따라 음악실로 이동하세요.',
  bidirectional: true,
}
```

앱 시작 시 `validateNavigationData`가 중복 ID, 없는 노드 참조, 잘못된 거리와 좌표 등을 검사합니다.

## 경로 탐색 방식

`src/algorithms/dijkstra.ts`의 `findShortestPath`가 아직 방문하지 않은 노드 중 누적 비용이 가장 낮은 노드를 반복 선택합니다. 각 간선의 `distance`를 기본 비용으로 사용하고 엘리베이터 대기, 경사도, 장애물, 휠체어 이동 시 불편한 폭에 페널티를 더합니다. 양방향 간선은 어느 방향에서도 탐색할 수 있습니다.

휠체어 모드에서는 다음 간선을 후보에서 제외합니다.

- `accessible`이 `false`인 통로
- 계단
- 장애물이 있는 통로
- 폭이 0.9m 미만인 통로
- 경사도가 8%를 초과하는 통로

일반 모드는 모든 간선을 사용할 수 있지만 장애물 등에는 비용이 추가되어 더 편한 경로가 우선될 수 있습니다.

## 프로젝트 구조

```text
src/
├── algorithms/      # Dijkstra와 테스트
├── components/      # 폼, 지도, 요약, 단계 안내 등 UI

├── main.tsx
└── styles.css
```

## 현재 구현의 한계

- 실제 GPS 위치를 사용하지 않습니다.
- 실제 학교 평면도와 정확히 일치하지 않습니다.
- 거리는 실제 미터가 아닌 상대적인 비용입니다.
- 사용자가 직접 현재 위치를 선택해야 합니다.
- 실제 장애물 상황은 실시간으로 반영되지 않습니다.
- 음성 품질과 제공되는 한국어 음성은 브라우저와 운영체제에 따라 다릅니다.

## 향후 개선 방향

- QR 코드 기반 현재 위치 확인
- 실제 거리 데이터 반영
- 학교 관리자용 데이터 수정 화면
- 장애물 신고 기능
- 실시간 엘리베이터 상태 반영
- 음성 인식 목적지 입력
